Homework 1: Tasks for onnn1

Your personal task is to develop a system for hairdressers and their stylists.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2123.hw1.domain.Stylist. With the following private fields:

  1. String name
  2. String qualification
  3. String expertise
  4. int price
Generate necessary getters and setters. [7 marks]

Create class co2123.hw1.domain.Hairdresser. With the following private fields:

  1. int id
  2. String city
  3. String founder
  4. List<Stylist> stylists
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2123.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Hairdresser> hairdressers in class co2123.hw1.Hw1Application. Initialize the list inside class co2123.hw1.Hw1Application to contain one hairdresser with two stylists. [4 marks]

Create controller co2123.hw1.controller.HairdresserController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/hairdressers" should show view hairdressers/list. [2 marks]
    • Add the list of all hairdressers from co2123.hw1.Hw1Application to the model as attribute hairdressers. [2 marks]
    • Create the corresponding JSP to list the city and founder of each hairdresser followed by a link to URL "/stylists?hairdresser={id of hairdresser}", e.g., "/stylists?hairdresser=3". [4 marks]
  2. Requests for URL "/newHairdresser" should show view hairdressers/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="hairdresser" and action="/addHairdresser" and inputs for all fields of class Hairdresser except for stylists. [5 marks]
  3. POST requests for URL "/addHairdresser" should add a new hairdresser to the list of all hairdressers in co2123.hw1.Hw1Application and redirect to /hairdressers. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2123.hw1.controller.StylistController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/stylists" should show view stylists/list. [2 marks]
    • Based on request parameter hairdresser determine the corresponding hairdresser from co2123.hw1.Hw1Application. [2 marks]
    • Add the list of all stylists of that hairdresser to the model as attribute stylists. [2 marks]
    • Create the corresponding JSP for the view to list the name, qualification, expertise, and price of each stylist. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newStylist?hairdresser={id of hairdresser whose stylists are listed}", e.g., "/newStylist?hairdresser=3". [4 marks]
  2. Requests for URL "/newStylist" should show view stylists/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="stylist" and action="/addStylist" and inputs for all fields of class Stylist. [6 marks]
    • Also add the request parameter hairdresser with the ID of the current hairdresser. [4 marks]
  3. POST requests for URL "/addStylist" should add a new stylist to the list of stylists of the hairdresser identified by request parameter hairdresser and then redirect to /hairdressers. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Hairdresser and make it available in controller HairdresserController. [4 marks]

When adding a new hairdresser validate that

  1. The value of field id is unique within the list Hw1Application.hairdressers. [3 marks]
  2. The value of field city is not empty. [2 marks]
  3. The value of field founder is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Stylist and make it available in controller StylistController. [4 marks]

When adding a new stylist validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field qualification is not empty. [2 marks]
  3. The value of field expertise is either "Bob cut", "Bowl cut", or "Buzz cut". [3 marks]
  4. The value of field price is between (including) 1 and 10. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
